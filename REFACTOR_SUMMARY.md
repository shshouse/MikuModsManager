# 模组管理器重构总结

## 文件夹结构重构

### 原始结构
```
/patch
  ├── {游戏名称}/
      └── 模组文件...
```

### 新结构
```
/games
  ├── {游戏名称}/
      ├── patch/
      │   ├── {模组1}/
      │   │   ├── 模组文件...
      │   │   └── readme.md (可选)
      │   └── {模组2}/
      │       ├── 模组文件...
      │       └── readme.md (可选)
      └── backup/
          ├── {时间戳1}/
          │   ├── install.log
          │   └── 备份的原始文件...
          └── {时间戳2}/
              ├── install.log
              └── 备份的原始文件...
```

## 新增功能

### 1. 模组安装功能
- **路径**: `/games/{游戏名称}/patch/{模组名称}/`
- **功能**: 
  - 扫描patch目录下的所有子文件夹作为可用模组
  - 安装时排除readme.md文件
  - 自动创建时间戳备份文件夹
  - 备份被替换的原始文件

### 2. 备份系统
- **时间戳格式**: `YYYY年MM月DD日_HH时MM分SS秒`
- **备份内容**:
  - 被替换的原始游戏文件
  - 中文安装日志(install.log)
- **日志内容**:
  - 游戏名称和模组名称
  - 安装时间戳
  - 每个文件的操作类型(新增/修改)
  - 影响的文件路径列表

### 3. 恢复功能
- **功能**: 读取备份文件夹中的install.log
- **恢复逻辑**:
  - 对于"修改"类型文件：从备份恢复原始文件
  - 对于"新增"类型文件：删除模组添加的文件
- **完整性**: 确保游戏状态完全还原到安装前

## 后端API更新

### 新增命令
1. `install_mod(mod_name, game_name, game_path)` - 安装模组
2. `get_available_mods(game_name)` - 获取可用模组列表
3. `get_backup_list(game_name)` - 获取备份记录列表
4. `restore_from_backup(game_name, timestamp, game_path)` - 从备份恢复

### 更新命令
1. `get_patch_folders()` - 更新为扫描新的games目录结构
2. `scan_installed_games()` - 使用新的create_game_folder函数

## 使用流程

### 安装模组
1. 将模组文件夹放置在`/games/{游戏名称}/patch/`目录下
2. 在应用中查看游戏详情
3. 在"可用模组"列表中点击"安装模组"
4. 系统自动备份原始文件并安装模组

### 恢复游戏状态
1. 在游戏详情页面查看"备份与恢复"列表
2. 选择要恢复的时间戳
3. 点击"恢复此状态"
4. 系统自动恢复到指定时间点的游戏状态
